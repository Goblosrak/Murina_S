Лабораторная работа №4: Синхронизация потоков
Выполнила: Мурина Софья К0709-23/3

Описание проекта

    Данный проект содержит реализации классических задач синхронизации потоков в C# для лабораторной работы 
    по изучению многопоточности, взаимоблокировок (deadlock) и методов их предотвращения.

Цели работы

    Изучить проблему синхронизации потоков
    Научиться выявлять и предотвращать взаимоблокировки (deadlock)
    Освоить использование примитивов синхронизации в .NET

Реализованные задачи
1. Обедающие философы (Dining Philosophers)
    
        Описание: 5 философов (потоков) делят 5 вилок (ресурсов)
        Реализовано:
        Версия с deadlock (демонстрация проблемы)
        Версия без deadlock (исправленная реализация)
        Метод предотвращения deadlock: Изменение порядка захвата вилок у одного философа
   
3. Спящий парикмахер (Sleeping Barber)
   
        Описание: Парикмахер спит, когда нет клиентов, и просыпается при их появлении
        Особенности: Ограниченная очередь клиентов (максимум 5 человек)
        Используемые примитивы: SemaphoreSlim, Mutex, Queue<T>

4. Производитель-Потребитель (Producer-Consumer)
   
        Описание: Производители добавляют товары в буфер, потребители забирают
        Требования: Не переполнять буфер, не пытаться взять из пустого буфера
        Реализация: Использование BlockingCollection<T>

Технологии и библиотеки

    .NET Core
    C# (.NET 8.0)
    System.Threading — базовые примитивы синхронизации
    System.Collections.Concurrent — потокобезопасные коллекции
